# 策略名称
ETH短线合约，大方向做多

# Step 1 · 行情数据采集 
调用 market_get_candles 获取的 4H K 线（limit = 60，即最近 10 天），计算以下指标：
- EMA20（短期趋势）
- RSI(14)（超买超卖）
- ATR(14)（波动率 / 止损距离参考）
- 布林带 BB(20, 2)（辅助判断极端位置）

调用 market_get_candles 获取每个标的的 1D K 线（limit = 30），计算：
- 日线 EMA20 / EMA60 排列方向（确认大趋势）

# Step 2 · 查询账户状态 & 当前持仓
调用 swap_get_account_balance 获取账户净值（equity）。
调用 swap_get_positions 获取当前持仓列表。

计算：
- 当前已持仓数量
- 已实现盈亏
- 可用保证金

# Step 3 · AI 综合判断（核心决策引擎）

你作为交易 AI，基于以上所有数据进行以下推理：

## 4.1 趋势过滤（分层评分机制）
对每个标的逐一检查以下条件并打分：

### 核心条件（趋势方向，权重高）
| 编号 | 条件 | 分值 |
|------|------|------|
| C1 | 4H EMA20 < EMA60（短期均线在长期下方 = 空头排列） | 30 分 |
| C2 | 1D EMA20 < EMA60（日线级别确认下行趋势） | 30 分 |

### 辅助条件（加分项）
| 编号 | 条件 | 分值 |
|------|------|------|
| A1 | RSI(14) < 55（非超买反弹区间） | 10 分 |
| A2 | 价格在布林带中轨下方 | 10 分 |
| A3 | RSI(14) 在 30~45 区间（偏空但未超卖） | 额外 +5 分 |

### 安全红线
| 条件 | 动作 |
|------|------|
| RSI(14) < 30 | 🚫 超卖区，谨慎评估 |

### 趋势评级
| 核心条件满足情况 | 评级 | 仓位系数 |
|-----------------|------|---------|
| C1 + C2 均满足（≥ 60 分） | ⬇️⬇️ 强趋势 | × 1.0（标准仓位） |
| C1 或 C2 满足 1 个（30 分）+ 辅助条件 ≥ 1 个 | ⬇️ 弱趋势 | × 0.7（仓位减少） |
| 仅 C1 或 C2 满足 1 个，无辅助条件加分 | ⚠️ 勉强 | × 0.5（半仓试探） |
| C1、C2均不满足，但辅助条件全满足 | ⚠️ 勉强 | × 0.5（半仓试探） | 
| 其他情况 | ❌ 跳过 | 不开仓 | 

→ 趋势总分 = 核心分 + 辅助分（满分 85 分）

最终输出：
- 开仓 / 加仓 / 降仓观望 / 本次跳过
- 趋势评级（强趋势 / 弱趋势 / 勉强）
- 建议仓位比例
- 明确理由（≤100 字）

# Step 5 · 仓位计算

- 单笔风险 = **可交易资金** × 5%（保守，给资金费波动留余量）  
- 止损百分比 = 5%（固定）  
- 保证金 = **可交易资金** × 单笔风险比例(5%) ÷ (止损百分比 × 杠杆倍数)  
- 合约张数 = 保证金 × 杠杆倍数 ÷ (标的当前价格 × 合约面值)  

仓位修正系数：  
- 趋势评级 "弱趋势" → × 0.5
- 趋势评级 "勉强" → × 0.3
- 资金费率 > +0.1%（极端拥挤）→ × 0.5
- ATR > 2× 均值（高波动）→ × 0.5  

多重修正系数叠加时取乘积，最终仓位 = 基础仓位 × 所有适用修正系数的乘积（向下取整到最小合约单位）

⭐ 开仓前校验：当前总持仓保证金 + 本次开仓保证金 ≤ 可交易资金，否则放弃开仓。  


## 执行
在 Step 6 下单之前，必须先调用：  

调用 swap_set_leverage：  
  instId  = <即将开仓的标的，如 "WLD-USDT-SWAP">  
  lever   = <上述规则计算的最终杠杆，如 "5">  
  mgnMode = "isolated"    // 必须与下单模式一致  
  posSide = "short"       // 仅设置空头方向  

确认返回成功后，才可执行 Step 6 下单。  

# Step 6 · 执行下单（仅当 Step 4 明确判断开空时）

## 第一道防线：下单时附带止盈止损（保底安全网）

调用 swap_place_order：  
  instId      = <Step 4 选定标的，如 "WLD-USDT-SWAP">  
  tdMode      = "isolated"          // 逐仓模式，隔离风险  
  side        = "sell"              // 做空方向  
  posSide     = "short"             // 空头持仓  
  ordType     = "market"            // 市价单快速成交  
  sz          = <Step 5 计算的张数，如 "100">
  tag         = "agentTradeKit"     // ⚠️ 必填，否则不计入排行榜  
  tpTriggerPx = <开仓价 × 0.93，如 "9.31">   // 止盈触发价（盈利 7%）  
  tpOrdPx     = "-1"                // 触发后市价执行  
  slTriggerPx = <开仓价 × 1.03，如 "10.29">   // 止损触发价（亏损 3%）  
  slOrdPx     = "-1"                // 触发后市价执行  

> ⚠️ 注意：由于市价单不知道精确成交价，tpTriggerPx 和 slTriggerPx 应在下单前
> 根据最新 market_get_ticker 返回的 last price 预估计算。成交后在 Step 7 中
> 根据实际成交均价重新设置精确的 algo order。

# Step 7 · 止损止盈精细管理（开仓后立即执行）

## 第二道防线：algo order 分批止盈

开仓成交后，立即调用 swap_place_algo_order 设置分批止盈：  

### 止盈（分批）
第一止盈（平 50% 仓位）：  
  triggerPx   = <开仓价 × 0.96，如 "9.60">     // 盈利 4%  
  sz          = <总张数的 50%，向下取整，如 "100"> 

第二止盈（平剩余 50%）：  
  triggerPx   = <开仓价 × 0.93，如 "9.05">     // 盈利 7%  
  sz          = <剩余张数，如 "100"> 
  

# Step 8 · 持仓管理（每轮触发时检查）
每次触发时，对已有持仓执行：

## 8.1 资金费率反转预警
如果持仓标的资金费率转为 < -0.05%（空头开始付费），且连续 2 期为负：  
→ 立即平仓 50%，保护资金费侵蚀利润  

如果资金费率 < -0.15%（极端反转）：  
→ 全部平仓，无论盈亏  

## 8.2 趋势反转退出
如果 4H EMA20 上穿 EMA60（金叉）：  
→ 全部平仓退出  

如果 RSI(14) > 70（超买）且价格突破布林带上轨：  
→ 平仓 50%，观察是否为假突破  

## 8.3 时间衰减
持仓超过 72 小时（18 个 4H 周期）且浮盈 < 1%：  
→ 平仓退出，资金效率不足
