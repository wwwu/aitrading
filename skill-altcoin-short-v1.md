# 策略名称
山寨币趋势做空 + 防资金费侵蚀 V1.0

# 策略理念
长期看空高 FDV 低流通山寨币，利用趋势信号确认 + 资金费率正向（多头付费给空头）窗口开空，赚取"趋势下跌 + 资金费率收息"双重收益。严格止损、绝不扛单。

# 监控标的池（AI 动态筛选）
WLD-USDT-SWAP   // Worldcoin — 高 FDV、解锁抛压大、流通 <20%  
APT-USDT-SWAP   // Aptos     — VC 高控盘、解锁周期长  
ARB-USDT-SWAP   // Arbitrum  — Layer2 叙事退潮、初期仅释放 15%  
DOGE-USDT-SWAP  // Dogecoin  — Meme 情绪驱动、退潮暴跌  

> 选币逻辑：市值/FDV 比 < 0.3、合约资金费率长期为正、近期有解锁事件优先。

# 执行节奏
每 1 小时触发一次

# ═══════════════════════════════════════════
# Step 1 · 行情数据采集
# ═══════════════════════════════════════════
对监控池内全部 4 个标的，分别调用：

调用 market_get_candles 获取每个标的 4H K 线（limit = 60，即最近 10 天），计算以下指标：
- EMA20（短期趋势）
- EMA60（中期趋势）
- RSI(14)（超买超卖）
- ATR(14)（波动率 / 止损距离参考）
- 布林带 BB(20, 2)（辅助判断极端位置）

调用 market_get_candles 获取每个标的的 1D K 线（limit = 30），计算：
- 日线 EMA20 / EMA60 排列方向（确认大趋势）

# ═══════════════════════════════════════════
# Step 2 · 资金费率 & 持仓量采集（核心风控信号）
# ═══════════════════════════════════════════
对每个标的调用：  

调用 market_get_funding_rate 获取当前资金费率 + 预估下费率。  
调用 market_get_funding_rate_history 获取最近 7 天资金费率历史（用于判断费率趋势）。  
调用 market_get_open_interest 获取持仓量变化。  

关键指标提取：  
- 当前资金费率 fundingRate
- 7 日平均资金费率
- 持仓量 24h 变化百分比

# ═══════════════════════════════════════════
# Step 3 · 查询账户状态 & 当前持仓
# ═══════════════════════════════════════════
调用 swap_get_account_balance 获取账户净值（equity）。
调用 swap_get_positions 获取当前持仓列表。

计算：
- 当前已持仓标的数量
- 当日已实现盈亏
- 可用保证金

# ═══════════════════════════════════════════
# Step 4 · AI 综合判断（核心决策引擎）
# ═══════════════════════════════════════════

你作为交易 AI，基于以上所有数据进行以下推理：

## 4.1 趋势过滤（必须全部满足才允许开空）
对每个标的逐一检查：  
□ 4H EMA20 < EMA60（短期均线在长期下方 = 空头排列）  
□ 1D EMA20 < EMA60（日线级别确认下行趋势）  
□ RSI(14) < 55（非超卖反弹区间；RSI < 30 时禁止追空，等反弹）  
□ 价格在布林带中轨下方  

→ 全部满足 = "趋势合格"  
→ 任一不满足 = 该标的本轮跳过  

## 4.2 资金费率评估
| 条件 | 动作 |
|------|------|
| 当前费率 > +0.01%（正费率，多头付给空头）| ✅ 利好做空，正常仓位 |
| 当前费率 > +0.05% | ⚡ 极度拥挤做多，做空最佳窗口，可满仓位 |
| 当前费率 > +0.1%（绝对值）| ⚠️ 过度拥挤，可能即将反转，仓位减半 |
| 当前费率 < -0.01%（负费率，空头付费）| 🚫 做空需付资金费，除非趋势极强否则跳过 |

## 4.3 波动率评估
| ATR 状态 | 动作 |
|----------|------|
| ATR(14) < 近 30 日 ATR 均值 × 0.5 | 波动过低，机会不足，跳过 |
| ATR(14) 在 0.5× ~ 1.5× 均值区间 | 正常，标准仓位 |
| ATR(14) > 近 30 日 ATR 均值 × 2.0 | 波动过大，仓位减半 + 止损放宽 |

## 4.4 持仓量异动
- 持仓量 24h 增加 > 20% 且费率为正 → 做多拥挤加剧，利好空头
- 持仓量 24h 减少 > 15% → 可能正在去杠杆，暂停新开仓

## 4.5 综合排名 & 最终决策
对所有"趋势合格"的标的按以下权重打分：
- 趋势强度（EMA 斜率）：40%
- 资金费率优势：30%
- 波动率合理性：20%
- 持仓量信号：10%

选出得分最高的 1~2 个标的（最多同时持有 2 个标的）。

最终输出：
- 开仓 / 加仓 / 降仓观望 / 本次跳过
- 选定标的 instId
- 建议仓位比例（占账户净值百分比）
- 明确理由（≤100 字）

# ═══════════════════════════════════════════
# Step 5 · 仓位计算
# ═══════════════════════════════════════════
- 单笔风险 = 账户净值 × 2%（保守，给资金费波动留余量）  
- 止损百分比 = 2%（固定）  
- 保证金 = 账户净值 × 单笔风险比例(2%) ÷ (止损百分比 × 杠杆倍数)  
- 合约张数 = 保证金 × 杠杆倍数 ÷ (标的当前价格 × 合约面值)  

仓位修正系数：  
- 资金费率 > +0.1%（极端拥挤）→ × 0.5
- ATR > 2× 均值（高波动）→ × 0.5
- 已持有 1 个标的 → 新开仓 × 0.7

最终仓位 = 基础仓位 × 修正系数（向下取整到最小合约单位）

# ═══════════════════════════════════════════
# Step 5.5 · 杠杆倍数设置（开仓前必须执行）
# ═══════════════════════════════════════════

## 杠杆倍数规则（硬编码，AI 不可自行调高）

| 标的类型 | 基础杠杆 | 最大允许杠杆 |
|----------|----------|-------------|
| DOGE（Meme 类） | 3x | 5x |
| WLD / APT / ARB（高 FDV 山寨） | 5x | 8x |

## 动态降杠杆条件
在基础杠杆上，遇到以下任一条件时必须降档：

| 条件 | 杠杆修正 |
|------|---------|
| ATR(14) > 近 30 日均值 × 2.0（高波动） | 基础杠杆 × 0.5（向下取整） |
| 资金费率 > +0.1%（极度拥挤，反转风险高）| 基础杠杆 × 0.5 |
| 已持有 1 个标的（分散风险） | 新标的杠杆 - 1 档 |

最终杠杆 = max(2, 修正后杠杆)，最低不低于 2x。

## 执行
在 Step 6 下单之前，必须先调用：  

调用 swap_set_leverage：  
  instId  = <即将开仓的标的，如 "WLD-USDT-SWAP">  
  lever   = <上述规则计算的最终杠杆，如 "5">  
  mgnMode = "isolated"    // 必须与下单模式一致  
  posSide = "short"       // 仅设置空头方向  

确认返回成功后，才可执行 Step 6 下单。  

## ⚠️ 杠杆铁律  
🔴 永远不允许杠杆 > 8x（山寨币波动极大，10x 以上一个插针即爆仓）  
🔴 DOGE 等 Meme 币永远不允许 > 5x  
🔴 AI 不可自主调高杠杆超过上表最大值  
🔴 杠杆设置必须在每次开仓前重新执行（防止手动修改后遗留高杠杆）  

# ═══════════════════════════════════════════
# Step 6 · 执行下单（仅当 Step 4 明确判断开空时）
# ═══════════════════════════════════════════
调用 swap_place_order：  
  instId  = <Step 4 选定标的，如 "WLD-USDT-SWAP">  
  tdMode  = "isolated"          // 逐仓模式，隔离风险  
  side    = "sell"              // 做空方向  
  posSide = "short"             // 空头持仓  
  ordType = "market"            // 市价单快速成交  
  sz      = <Step 5 计算的张数>  
  tag     = "agentTradeKit"     // ⚠️ 必填，否则不计入排行榜  

# ═══════════════════════════════════════════
# Step 7 · 止损止盈设置（开仓后立即执行）
# ═══════════════════════════════════════════
开仓成交后，立即调用 swap_place_algo_order 设置止损止盈：  

## 止损（硬性，不可取消）  
  instId      = <开仓标的>  
  tdMode      = "isolated"  
  side        = "buy"             // 空头平仓方向  
  posSide     = "short"  
  ordType     = "trigger"  
  triggerPx   = 开仓价 × 1.02     // 2% 止损  
  orderPx     = "-1"              // 触发后市价执行  
  triggerPxType = "last"  
  sz          = <全部持仓张数>  

## 止盈（分批）
第一止盈（平 50% 仓位）：  
  triggerPx   = 开仓价 × 0.97     // 盈利 3%  

第二止盈（平剩余 50%）：  
  triggerPx   = 开仓价 × 0.94     // 盈利 6%  

## 移动止损（当浮盈 > 3% 时启用）  
  当价格触及开仓价 × 0.97 后，将止损移至开仓价 × 1.00（保本位）  
  当价格触及开仓价 × 0.94 后，将止损移至开仓价 × 0.97（锁定 3%）

# Step 8 · 持仓管理（每轮触发时检查）
# ═══════════════════════════════════════════
每次触发时，对已有持仓执行：

## 8.1 资金费率反转预警
如果持仓标的资金费率转为 < -0.03%（空头开始付费），且连续 2 期为负：  
→ 立即平仓 50%，保护资金费侵蚀利润  

如果资金费率 < -0.1%（极端反转）：  
→ 全部平仓，无论盈亏  

## 8.2 趋势反转退出
如果 4H EMA20 上穿 EMA60（金叉）：  
→ 全部平仓退出  

如果 RSI(14) > 70（超买）且价格突破布林带上轨：  
→ 平仓 50%，观察是否为假突破  

## 8.3 时间衰减
持仓超过 72 小时（18 个 4H 周期）且浮盈 < 1%：  
→ 平仓退出，资金效率不足
